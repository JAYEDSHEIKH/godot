NEX Scripting Language

modules/nexscript/ — drop-in Godot module

A statically-typed, immutable-by-default scripting language that compiles to a typed IR and executes on a dedicated typed-slot VM — designed to be faster than GDScript while remaining Godot-native.

  • Recursive-descent parser with full AST representation
  • Static type system — Int, Float, Bool, String, Vector2/3/4, Color, Nil, Any
  • Immutable-by-default bindings (let); mutable with var
  • Typed IR with SSA-style slots and jump tables
  • Switch-dispatch VM with typed operand stack (no boxing for primitives)
  • Result<T, E> propagation & range-based for-loops in IR
  • Semantic analyzer with type inference & constant folding optimizer
  • Full GDScript interop via NexLanguage / NexScript / NexInstance

Godot AI Agent

modules/ai_engine/ — in-editor AI copilot

A first-class AI assistant baked into the Godot editor. Supports multiple LLM providers, tool-execution against the live scene/project, RAG over the project codebase, and a persistent memory layer across sessions.

  • Providers: OpenAI (GPT-4o/o1) & Google Gemini with streaming
  • Tool execution — scene editing, script gen, asset search, shell, git, debug
  • Vector DB with cosine/dot/Euclidean similarity for RAG
  • Embedding chunker + project indexer for full codebase search
  • Session & long-term memory manager with configurable retention
  • Advanced prompting — chain-of-thought, few-shot, structured output
  • Scene & code analyzers for deep project context
  • Metrics, rate limiter, LRU cache, structured logger

AI Copilot Editor Dock

editor/docks/ai/ — integrated into EditorNode

A native Godot editor dock registered as an EditorDock subclass. Docked to the right panel by default, floatable, and fully theme-aware. The dock surfaces all AI engine capabilities in a tabbed UI.

  • Chat panel — markdown-rendered conversation with streaming output
  • Diff viewer (AIDiffViewer) — side-by-side code change review
  • Memory panel — browse and manage AI memory across sessions
  • Tools panel — enable/disable tool categories per session
  • Settings panel — provider key, model, temp, context window
  • Status bar — live provider state & token usage
  • Registered in editor_node.cpp & register_editor_types.cpp
  • SCsub picks up docks/ai/*.cpp automatically

Build System Integration

SCsub · config.py · register_types

Both modules plug into the standard Godot SCons module pipeline. No patches to engine core were required — only the EditorNode and type-registration files were extended.

  • modules/nexscript/config.py — can_build always true
  • modules/ai_engine/config.py — can_build always true
  • Both SCsub files recurse into all subdirectories
  • editor/editor_node.h/.cpp — AIDock member + initialization
  • editor/register_editor_types.cpp — AIDock & AIDiffViewer registered
  • editor/docks/SCsub — ai/*.cpp glob added
  • AI Dock placed in default layout at DOCK_SLOT_RIGHT_BL
  • Layout key "AIAssistant" for persistent dock position
